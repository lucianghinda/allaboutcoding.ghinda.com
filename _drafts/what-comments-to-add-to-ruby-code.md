We have this saying in the Ruby community that is something like:

* Good code does not need comments or maybe "[Eloquent code needs no comment"](http://alexis.bernard.io/blog/2023-08-11-eloquent-code-needs-no-comment.html)
    

And I do agree that because Ruby is so expressive it allows to write code in such a way that it mostly don't require comments.

But maybe, there are cases where a comment will go long way.
